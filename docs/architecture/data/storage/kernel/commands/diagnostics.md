# Diagnostics Commands

## Purpose

Diagnostics command metadata covers relay metadata, relay summaries, NIP-65
suggestions, author routes, protected route blocks, notifications, jobs, and app
log rows. Diagnostics use real rows or explicit unavailable states; product UI
must not synthesize relay metadata or notification previews.

## Safety Rules

Relay diagnostics, suggestions, routes, and finished jobs are ledger-backed
recoverable diagnostics or cache rows. App log rows are bounded diagnostics rows
without a current ledger step. Route blocks are protected safety rows, not
ledger-backed rows, and retention must never prune them.
