# Protected Commands

## Purpose

Protected command metadata covers user-owned rows that cache pressure must never
prune. Tab snapshots are dynamically protected while active and remain ledger
tracked only for stale snapshot cleanup.

## Common Proof

Focused tests: `cargo test -p lkjstr-storage commands`, protected row tests, and
`cargo test -p lkjstr-web storage` when worker adapters change. Deletion
condition for every row: Rust product surface parity, ledger update where
needed, focused tests, and no TypeScript import proof.

## Route Blocks

Route-block commands live in [diagnostics.md](diagnostics.md) because the worker
adapter is part of relay route diagnostics. Their protection policy is still
`protected`, their data class is `protected-safety-configuration`, and retention
must never prune them.
