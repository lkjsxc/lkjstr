# SQLite Worker Protocol

## Purpose

This file defines the typed protocol between Rust/WASM and the SQLite worker.
Status: design target.

## Request Envelope

Each request carries:

- `request_id`: stable id generated by the host adapter.
- `deadline_ms`: caller deadline relative to request dispatch.
- `op`: one storage operation.

Operations:

- `Open`: database name, VFS preference, and startup flags.
- `Close`: close the database and release callbacks.
- `ApplySchema`: schema hash and ordered statements.
- `Execute`: statement id, SQL text, and parameters.
- `Query`: statement id, SQL text, parameters, and row limit.
- `Batch`: transaction mode and ordered SQL steps.
- `EstimateStorage`: browser usage and quota diagnostics.
- `Compact`: target bytes and protected owner keys.
- `ExportJson`: explicit user-selected data classes.
- `ImportJson`: explicit user-provided payload.
- `Cancel`: request id to cancel.

## Response Envelope

Each response carries:

- `request_id`.
- `outcome`.
- rows.
- affected row count.
- storage diagnostics.

Outcomes:

- `Ok`.
- `Unavailable`.
- `Timeout`.
- `Busy`.
- `Blocked`.
- `Quota`.
- `Corrupt`.
- `Canceled`.
- `LateSettled`.
- `LateRejected`.

## Cancellation

Cancellation is explicit. The app cancels storage work when a tab closes, a feed
runtime releases demand, a publish job is abandoned, or a deadline expires.

The host adapter clears callbacks on cancel and close. A late worker response is
not delivered to product logic; it increments diagnostics and returns a late
outcome to Stats.

## Timeouts

The host adapter owns request deadlines. The worker should also observe
deadlines before starting large batches, but the app does not rely on the worker
for cleanup. Deadline expiration returns `Timeout` and drops the callback owner.

## Row Values

Rows cross the worker boundary as typed scalar values and JSON text payloads.
The worker does not infer product types from JSON. Rust repositories decode row
fields into domain records.
