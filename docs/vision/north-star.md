# North Star

Build a repository that is easy to understand, easy to change, and explicit
about its current limits.

The project should favor:

- Clear documentation before broad implementation.
- Small files with focused responsibilities.
- Local verification that can be repeated by any contributor.
- Decisions that are written down close to the work they guide.

## Success Shape

A contributor should be able to open the repository, read the documentation, and
understand what exists, what is missing, and what checks are expected before
handoff.
